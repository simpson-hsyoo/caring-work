# doc-summary

문서를 구조화된 형태로 요약하는 스킬.

## 기저 원칙

- **원문 불변**: 원본 텍스트는 절대 수정/축약하지 않는다.
- **분리 원칙**: Claude 산출물은 항상 원본과 별도 영역에 작성한다.

---

## Core Workflow

### 0. 목적 수집 (요약 전 필수)

아래 항목을 사용자에게 질문한다. 모호한 답변에는 선택지를 제시하며 구체화될 때까지 반복한다.

- 요약 목적 (요구사항 관리, 회의록 정리, 의사결정 추적 등)
- 기대 산출물 형태 (DB, 표, 텍스트 요약 등)
- 포함/제외 기준 (특정 섹션만, 참고자료 제외 등)
- 작업 위치 — **AskUserQuestion으로 선택형 제시** (예: 원본에 직접 추가 / 개인 영역에 복사 후 작업)

### 1. 원문 분석 + 초안 설계

- 문서의 섹션/항목/분류 체계를 파악한다.
- 0단계에서 수집한 목적을 바탕으로, 아래를 포함한 초안을 설계하여 제시한다:
  - 항목 단위 (문장/문단/섹션 중 적절한 수준)
  - 산출물 형태 (DB / 표 / 텍스트)
  - 스키마 (속성 목록, 분류 체계). No(순번)는 필수 포함.

### → 사용자 검토 (게이트)

- 초안을 사용자에게 제시하고 승인/수정을 받는다.
- 확정될 때까지 다음 단계로 진행하지 않는다.

### 2. 항목 추출

- 원문을 그대로 보존하여 추출한다. 수정/축약 금지.
- 원문 등장 순서대로 No를 부여한다 (1부터 연번).

### 3. 주석/댓글 이관

- 원본의 댓글/주석을 해당 항목에 삽입한다.
- **원문 + 요약 병기**: 원본 댓글(작성자, 날짜 포함)을 그대로 보존하고, 별도 영역에 Claude 요약본을 함께 작성한다.

```
── 댓글 ──
[요약] (Claude가 작성한 댓글 요약)
[원문]
- 작성자A (날짜): "댓글 원문..."
- 작성자B (날짜): "댓글 원문..."
```

### 4. 제외 항목

- 데이터베이스/산출물에 포함하지 않은 원문 내용을 정리한다.
- 각 항목별로 무엇을, 어디에 있었고, 왜 제외했는지 명시한다.

### → 원문 검증 (게이트)

별도 에이전트로 원문 보존 여부를 검증한다. 글자/항목/형태 3관점 점검.
위반 시 자동 수정 후 재검증, 전건 통과 시 다음 단계로 진행.
상세: [agents/text-verification.md](agents/text-verification.md)

### 5. 산출물 포장

- Claude 생성물임을 식별 가능하게 래핑한다.
- 원본 콘텐츠와 명확히 분리된 형태로 배치한다.
- 구체적인 래핑 방식은 Tool Addon을 따른다.
