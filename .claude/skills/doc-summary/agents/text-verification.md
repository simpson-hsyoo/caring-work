# Agent: 원문 검증

2~4단계 산출물을 도구에 반영하기 전에, 추출 작업 컨텍스트와 독립적으로 원본을 다시 읽고 비교한다.

## 검증 기준

| 관점 | 확인 내용 | 위반 예시 |
|------|-----------|-----------|
| 글자 단위 | 추출된 "원문" 필드가 원본 텍스트와 일치하는가 | 요약문 대체, 단어 삭제/변경, 문체 변경 |
| 항목 단위 | 원본의 모든 항목이 빠짐없이 추출되었는가 | 항목 누락, 중복 추출, 제외 항목과의 불일치 |
| 형태 단위 | 계층 구조가 유지되었는가 | 불릿/번호 제거, 들여쓰기 평탄화, 섹션 제목 누락 |

## 실행 방식

1. 항목을 배치로 분할하여 병렬 에이전트로 점검한다.
2. 각 에이전트는 원본 문서를 독립적으로 다시 읽고, 산출물의 원문 필드와 1:1 비교한다.
3. 위반 발견 시: 불일치 목록(No, 원본, 현재값, 위반 유형)을 보고하고, 자동 수정 후 재검증한다.
4. 전건 통과 시: 다음 단계(산출물 포장)로 진행한다.

## 설계 근거

같은 컨텍스트에서 self-check를 수행하면 동일 편향이 반복될 수 있다.
별도 에이전트가 원본을 처음부터 다시 읽는 구조가 위반 검출 신뢰도를 높인다.
