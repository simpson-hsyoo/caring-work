# 8. 회선 단독 회수

## 시나리오 정의

| 항목 | 내용 |
|------|------|
| 트리거 | 회선 단독 회수 필요 |
| 행위자 | 회수자 (총무F) |
| 입력 | 회수 대상 매핑, 회수 후 처리 방향 |
| 출력 | 회선↔owner 매핑 해제<br/>+ 회선 상태 변경<br/>+ 이력 |
| 사전조건 | 회선↔owner 매핑 활성 |
| 사후조건 | 매핑 해제 |
| 연관 카테고리 | [7](07-회선단독배정.md) (재배정), [4](04-회선해지.md) (해지) |

## Step 시퀀스

| # | 행위자 | 행위 | 분기/예외 |
|---|--------|------|-----------|
| 1 | 회수자 | 회수 트리거 진입 | — |
| 2 | 회수자 | 회수 대상 매핑 선택 | — |
| 3 | 회수자 | 회수 후 처리 선택 | 재고화 / 재배정 / 회선 해지 |
| 4 | 회수자 | 부가서비스 처리 | — |
| 5 | 시스템 | 회선↔owner 매핑 해제<br/>+ 회선 상태 변경<br/>+ 이력 기록 | — |

## Mermaid Flowchart

```mermaid
flowchart TD
    Start([회선 단독 회수 시작]) --> Trigger[회수 트리거 진입]
    Trigger --> Target[회수 대상 매핑 선택]
    Target --> Post{회수 후 처리}
    Post -->|재고화| Stock[본사 재고로]
    Post -->|재배정| ReAssign[시나리오 7로]
    Post -->|해지| Cancel[시나리오 4로]
    Stock --> Svc
    ReAssign --> Svc
    Cancel --> Svc
    Svc[부가서비스 처리] --> Release[회선↔owner 매핑 해제<br/>+ 회선 상태 변경<br/>+ 이력 기록]
    Release --> End([회수 완료])
```
