# CCX (Caring Call eXperience)

케어링 콜 관리 솔루션. CTI 내재화 프로젝트.

## 비즈니스

- **정의**: 전사 콜 관리 솔루션 (본사CTI · 콜매니저 · TLINK · Slack 워크플로 통합)
- **목적**: 외부 솔루션 의존 해소, 데이터 통합, 운영 효율화
- **단계**: 기획·설계 단계 (As-Is 도메인 파악 완료, To-Be 유저 플로우 설계 중)

## As-Is 구성

| 시스템 | 사용 주체 | 역할 |
|--------|----------|------|
| 본사 CTI (caring.codehive.co.kr) | 성장F 컨택팀 | 본사 콜 상담 |
| 콜매니저 (caringbranch.codehive.co.kr) | 본부 지점 | 지점 콜 관리 |
| TLINK | 성장F 성장기획팀 | 콜 데이터 확인·녹취 |
| Slack 워크플로 | 본부/센터 | 상담 배정·상담시트 접수 |
| Google Sheet | 본부/센터 | 주간보호 상담시트 |

## 외부 의존성

```
KT (번호 발급) → 드림라인 (회선·단말 영업·번호이동·DID) → CNAI (SIP·내선·매개번호) → TLINK (앱·녹취)
```

- 회선 한 개에 외부 업체 다단계 종속: 대표번호 → 드림라인 매개번호 → CNAI 매개번호 → 내선번호 (1:1:1:N)
- 주말/공휴일 외주 대응 어려움 (총무F 페인포인트)
- TLINK 주소록 파편화, 매핑 휴먼 에러 다발
- 에이닷전화 등 비기본 통화앱 사용 시 녹취 미생성

## 이해관계자

- **성장F 컨택팀** — 본사 콜 상담 (본사CTI 사용)
- **성장F 성장기획팀** — 콜 데이터 분석 (TLINK 사용)
- **본부/센터** — 지점 콜 관리·송영 (콜매니저·법인폰 사용)
- **총무F** — 자산(전화 단말기·법인폰) 관리, 외부 통신사 소통
- **사용자(요보사/사회복지사·송영)** — 법인폰 사용 주체

## 객체 모델 (자산관리 영역 도출)

| 객체 | 의미 | 비고 |
|------|------|------|
| **자산** | 케어링 보유 모든 물리 자산 (전화 단말기·차량·가구 등) | 자산 마스터에 등록 |
| **전화 단말기** | 자산의 하위 종류. IP폰·법인폰(010) | 회선과 1:1 매핑 |
| **회선** | 통신 회선 (번호·DID·내선) | 외부 통신사 발급 |
| **owner** | 자산 사용 주체 | 사용자 / 조직 공용 / 미지정(=재고) |
| **소속 조직** | 자산이 속한 조직 | 본사 / 본부 / 센터. 모든 소속에서 재고(미지정) 가능 |

핵심 매핑:
- `자산↔소속 조직` (모든 자산)
- `자산↔owner` (사용 주체)
- `전화 단말기↔회선` (단말기당 1회선 원칙)
- `전화 단말기↔회선↔owner` (배정 시 3-way)

## 진행 중 작업

### 자산관리 유저플로우 (`asset-mgmt/`)

8개 카테고리 시나리오 작성. **1~4번 검토 완료, 5~8번 검토 보류 중.**

| # | 카테고리 | 상태 |
|---|----------|------|
| 1 | 신규 자산 취득 | ✅ 검토 완료 |
| 2 | 자산 폐기 | ✅ 검토 완료 |
| 3 | 신규 회선 등록 | ✅ 검토 완료 |
| 4 | 회선 해지 | ✅ 검토 완료 |
| 5 | 전화 단말기·회선 배정 | ⏸ 검토 중 (포인트 5개 제안 상태, 사용자 답변 대기) |
| 6 | 전화 단말기·회선 회수 | ⏸ 미검토 |
| 7 | 회선 단독 배정 | ⏸ 미검토 |
| 8 | 회선 단독 회수 | ⏸ 미검토 |

#### 5번 검토 포인트 (재개 시 시작점)

1. 전화 단말기 선택 조건 (회선 미부여 한정 같은 명시)
2. 회선 선택 조건 (미배정만 / 3번에서 매핑된 회선도 가능?)
3. 3번 등록 시 이미 매핑된 케이스의 5번 시나리오 처리
4. 매핑의 도메인 의미 (3-way 통합 vs 3개의 2-way 조합)
5. 부가서비스 설정 처리

## 도메인 특화 용어 원칙

플로우차트 작성 원칙은 [`.claude/skills/make-flow-chart/SKILL.md`](/.claude/skills/make-flow-chart/SKILL.md) 참고. 추가 CCX 특화:

- **"전화 단말기"** 용어 사용 (모호한 "단말기" 대신)
- **"owner"** 용어 사용 (주체·사용자 혼용 금지)
- **매핑 표기** 시 무엇과의 매핑인지 반드시 명시
- 3-way 매핑은 `전화 단말기↔회선↔owner` 순서로 통일
- 2-way 매핑은 `회선↔owner` (자산 폐기·해지에서 사용)
