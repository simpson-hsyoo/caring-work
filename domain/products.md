# 케어링 제품 구성

요양시스템F가 담당하는 제품군. 장기요양 서비스의 라이프사이클에 따라 역할이 나뉜다.

```
계약 前                 계약 後 (서비스 제공)      전 과정
┌──────────┐      ┌──────────────┐      ┌──────────┐
│  CAATS   │  →   │     ERP      │      │  NEXUS   │
│ 상담·매칭 │      │  행정·운영   │      │  플랫폼  │
└──────────┘      └──────────────┘      └──────────┘
                                         ┌──────────┐
                                         │   CCX    │
                                         │유선 상담 │
                                         └──────────┘
```

---

## NEXUS (플랫폼)

- **역할**: 전 제품을 관통하는 플랫폼 레이어
- **핵심 기능**:
  - 권한 모델 관장 (본부 조직 계층 기반, 계층적 누적)
  - 데이터 표준화 — 회원, 부서, 행정구역(좌표→법정동 변환) 등 마스터 데이터
  - 보안 — RBAC 모듈, 다른 제품(cms-api 등)이 라이브러리로 참조
  - 센터별 지역 커버리지 관리
  - 수급자 통계
- **사용자**: 전 직군 (플랫폼 레이어로서 전 제품이 의존)
- **프론트엔드**: caring-web-monorepo `apps/nexus/` — 본사 지표 대시보드
- **백엔드**: nexus-api (Kotlin + Spring Boot)
- **데이터**: 조직 구조, 사용자 계정, 권한 정책, 행정구역, 수급자 통계

---

## CAATS (상담·매칭)

Customer Acquisition & Activation Tracking System. 백엔드 레포명은 cms-api.

- **역할**: 계약 전 단계 — 수급자·요양보호사 유입부터 매칭·계약 체결까지
- **핵심 기능**:
  - 수급자 상담 접수·관리
  - 요양보호사 구인·구직 관리
  - 수급자-요양보호사 매칭 (구인·구직 포함, 매칭 진행 상태 추적)
  - 계약 성사 여부 추적 (실제 계약 체결은 ERP에서 진행)
  - 전자문서·팩스 연동
- **사용자**: 사회복지사, 센터장, 본부장·본부관리팀(모니터링)
- **프론트엔드**: caring-web-monorepo `apps/caats/` (Next.js, Swagger 타입 자동생성)
- **백엔드**: cms-api (Kotlin + Spring Boot)
- **데이터**: 수급자 정보, 상담 기록, 구인·구직, 매칭 이력

---

## ERP (행정·운영)

- **역할**: 서비스 제공 중/후 단계 — 급여 제공 기록, 인력 관리, 청구
- **핵심 기능**:
  - 급여 제공 기록 관리 (업무수행일지, 급여제공계획서)
  - 요양보호사 스케줄·근태 관리
  - 공단 청구 관련 데이터 관리 (급여 실적, 보험)
  - 계약 관리 (수급자-센터 간)
  - 전자문서·전자서명
  - 요양보호사 보수교육 이수 관리
  - 알림·발송 (SMS/MMS)
  - 기관 평가 대응 — 평가 지표 산출에 필요한 데이터 관리
- **사용자**: 사회복지사, 센터장, 본부관리팀, 행정마스터
- **프론트엔드**: caring-web-monorepo `apps/erp/` (React/CRA)
- **백엔드**: erp-api (Kotlin + Spring Boot)
- **데이터**: 급여 실적, 인력 정보, 계약, 스케줄, 청구 내역

### ERP 주요 도메인 패키지

| 패키지 | 영역 |
|--------|------|
| recipient, recipient_ia | 수급자, 수급자 개인별 장기요양 |
| employee | 요양보호사·직원 |
| contract | 계약 |
| schedule | 스케줄 |
| salary | 급여 |
| visit | 방문 기록 |
| evaluation | 업무수행일지 급여반영 여부, 보수교육 |
| insurance | 보험 |
| edoc, esign | 전자문서, 전자서명 |
| send, allim | 발송, 알림 |
| consult | 상담 (상담 평가 포함) |
| approval | 결재 |
| automation | 자동화 연동 |
| external | 외부 시스템 연동 |

## CCX (유선 상담 관리)

- **역할**: 센터의 유선 상담(전화) 채널 통합 관리
- **핵심 기능**:
  - CTI 솔루션 내재화
  - 콜 상태 기록·녹취 (응대/부재중/호전환/착신전환)
  - 상담사 상태 관리, 후처리(WRAP_UP)
  - 녹취 파일 모니터링
- **사용자**: 고객컨택팀(인바운드·아웃바운드), 주간보호센터, 방문요양센터, 본사 어드민
- **프론트엔드**: ccx-web (React + TypeScript, PoC) → caring-web-monorepo 합류 예정
- **백엔드**: caring-ccx (Kotlin + Spring Boot, 주 개발 레포)
- **현재 상태**: 개발 중 (caring-ccx 기준 단위 137건 + 통합 87건 테스트)
- **장기 계획**: CAATS-ERP와 연동 예정

## 프론트엔드 구조

전 제품의 웹 프론트엔드를 **caring-web-monorepo** (Yarn v4 모노레포)에서 통합 관리한다.

| 앱 | 경로 | 프레임워크 | 대응 백엔드 |
|----|------|-----------|------------|
| CAATS | `apps/caats/` | Next.js | cms-api |
| ERP (통합요양) | `apps/erp/` | React (CRA) | erp-api |
| NEXUS (본사 지표) | `apps/nexus/` | Next.js | nexus-api |
| 주간보호 | `apps/daycare/` | React | — (미구현) |
| 자동화 관리 | `apps/auto-admin/` | — | — |
| 디자인 시스템 | `apps/design-system-docs/` | Storybook | — |

공유 패키지: `@care-kit/design-system`, `@care-kit/form`, `@care-kit/hooks`, `@care-kit/util`

## 공단포털 연동

- ERP/CAATS의 데이터 최신성이 공단포털(노인장기요양정보시스템)에 종속
- 수집 방식: Playwright UI 자동화 + API 병행
- 수집 데이터: 인정서 정보, 등급 변경, 급여 실적 등 (상세 범위는 공단포털 연동 문서에서 별도 정리 예정)
- 인증: 기관 단위 공동인증서
- 자동화 레포: erp-webscraper (Playwright), CaringAutomation (Python/Flask)
